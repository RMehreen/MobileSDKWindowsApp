Conversion routines

Convert between one string type and another

Sometimes it is necessary to convert a string before it can be used with the API. Sometimes it is also necessary to convert something that is returned. The most obvious need to convert comes with multiple string support. The three different types of strings are:

  • UTF-8
  • UTF-16
  • BSTR

BSTR and UTF-16 are both 16-bit Unicode but are stored in slightly different ways. UTF-8 encapsulates anything UTF-16 or BSTR can represent. Internally, the code uses UTF-8 extensively since it is better suited for transport over the network and also is better supported by the Mobile Receivers.

Internally, the Mobility Pack uses a COM object to support the API. This makes it necessary to convert to BSTR for the sake of COM automation. The application does not need to worry about this conversion unless it is programming directly to the COM interface.
