Location Services

The interfaces provided for location services within applications are those provided by the Windows 7 Location API.

The details of this API can be found on the Microsoft Developer Network (MSDN), and the Microsoft Windows SDK, from version 7.1 onwards. If you do not have a version of the MSDN library or Microsoft Windows SDK documentation installed, you can find this at http://msdn.microsoft.com/en-us/library/dd464636%28v=VS.85%29.aspx

The MSDN and Microsoft Windows SDK provide:

  • Documentation of the Location API calls.
  • Sample applications.
  • Headers and include files needed for native code linkage.

Natively, Windows 2008 R2 does not provide Windows Location API support. It is only with the Mobility Pack that this API is available on XenApp. When developing location aware applications to use with XenApp, please consider the following:

  • Location information will not be retrieved from client devices and available for applications unless two layers of policies and permissions are satisfied:
    • At the XenApp side, the Citrix user policy setting ICA > Client Sensors > Location must be set to Allowed.
    • At the Citrix Receiver side, when an application attempts to retrieve location information for the first time in a session, the user will be asked if they want to allow location information to be provided.
    • The user decision as to whether to allow location information to be available or not at the Citrix Receiver side takes precedence over the XenApp side Citrix user policy setting.
  • A location enabled application must not depend solely on location.
    • A user might not allow access to location information.
    • The location might not be available or might change while the application is running.
    • A user might connect to the application session from a different device that does not support location information.
  • A location enabled application must:
    • Have the location feature turned off by default.
    • Provide a user option to allow / disallow the feature while the application is running.
    • Provide a user option to clear location information cached by the application. Receiver does not cache location data.
  • A location enabled application must mange the granularity of the location information so that the data acquired is appropriate to the purpose of the application and conforms to regulations in all relevant jurisdictions.
  • A secure connection via SSL or VPN is recommended when using location services.
  • By default, location aware applications are configured to use LOCATION_DESIRED_ACCURACY_DEFAULT, which is an accuracy optimized for power use and other cost considerations. If you want to deliver the most accurate location possible, use an accuracy of LOCATION_DESIRED_ACCURACY_HIGH. This accuracy level is set with the ILocation.SetDesiredAccuracy() method in the COM Interface, and set with the DesiredAccuracy property for the LatLongReportFactory and CivicAddressReportFactory interfaces in the Location API scripting object model.
  • There are two mechanisms for retrieving location information; explicit retrieval of location by requesting a location report, and registering for location update events. In order to achieve the most accurate location information, register for location update events, which can provide more accurate information than that provided by an explicit location report request.
