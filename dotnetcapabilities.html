Feature Capabilities

C# Feature Capabilities sample program

Source: csharp/capabilities/Program.cs

The idea of supported capabilities stems from the concept of negotiated feature sets between client and server. The resulting set of supported capabilities can be determined using SDK API.

This sample program retrieves the different capabilities available. Review the Feature Capabilities for more detail on the concept.

It is important to remember that every request for a capability requires a capability Id CMP_CAP_ID and a key Id for the requested capability.

The type of the capability determines the value format. At this time, capabilities only include numeric values.

Note that these information are initialised upon session connection, and will remain unchanged for the duration of the session. For example, the display width and height reflect the resolution at the time of the session connection, and these values will not change regardless of orientation of the device during this session.

Below are examples of how to retrieve capability values of the connected device. 

        // <summary>
        // Display bool capability value.
        // </summary>
        // <param name="capId">Capability ID.</param>
        // <param name="keyId">Capabiluty Key ID.</param>
        // <param name="prefixOutput">Display prefix.</param>
        // <returns>CMPRESULT return code.</returns>
        static CMPRESULT GetCapBool(CMP_CAP_ID capId, short keyId, string prefixOutput)
        {
            CMPRESULT rc;
            bool keyValue;

            rc = cmp.GetCapabilityBool(capId, keyId, out keyValue);

            if (CMP_SUCCESS(rc))
            {
                Console.WriteLine("{0} {1}", prefixOutput, keyValue);
            }
            else
            {
                Console.WriteLine("{0} Error({1:X})", prefixOutput, rc);
            }

            return (rc);
        }
