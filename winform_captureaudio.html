Capture Audio

WinForm Capture Audio sample program

This sample illustrates how to capture audio from the connected device, and how to subscribe to the AudioCaptured event.

For Capture Audio to work, please ensure that Client Drive Mapping (CDM) is enabled, and full access is granted for the SD card on the connected device.

Source

winform/captureaudio/Form1.cs

The cmp referenced in the code snippets below is an instance of the CitrixMobile class.

CaptureAudio

The following is an example of how to capture audio from the device.

options is a CMP_CAPTURE_AUDIO_OPTIONS structure used to specify the desired options for the captured audio, this structure must be initialised before calling CMPCaptureAudio.

uniqueId is an unique identifier returned from the CaptureAudio call. This identifier will be referenced in a subsequent AudioCaptured event when the audio is acquired.

                    // Request a audio capture.
                    int rc = cmp.CaptureAudio(ref options, out captureId);

Event subscription

The CitrixMobile class exposes an event called CMPCaptureAudio.

When the CMPCaptureAudio request is fulfilled by the connected device, the application will be notified via a AudioCaptured event when the audio is ready.

The following is an example of how to subscribe to the AudioCaptured event.

                    // Register for AudioCaptured event.
                    cmp.AudioCaptured += new ICMPEvents_AudioCapturedEventHandler(cmp_AudioCaptured);

Event handler

The following is an implementation of the AudioCaptured event handler.

uniqueId is an unique identifier returned in a preceding CMPCaptureAudio request.
Filename is the location of the audio, this is provided as an UNC path which corresponds to the Client Drive Mapping for the connected device.

        // <summary>
        // AudioCaptured event handler.
        // </summary>
        // <param name="rc">Capture result.</param>
        // <param name="uniqueId">Capture identifier.</param>
        // <param name="audioMetadata">Capture metadata filename.</param>
        // <param name="filename">Capture filename.</param>
        // <param name="audioSize">Capture size.</param>
        // <param name="Quality">Capture quality.</param>
        // <param name="Encoding">Camera encoding.</param>
        // <param name="DurationLimit">Capture duration limit.</param>
        void cmp_AudioCaptured(CMPRESULT rc, CMP_UNIQUE_LONG_ID uniqueId, string audioMetadata, string filename, 
            CMPRESULT audioSize, CMP_CAPTURE_QUALITY Quality, CMP_AUDIO_ENCODING Encoding, CMPRESULT DurationLimit)
        {
            // only process capture if it is one of ours
            if (uniqueId == captureId)
            {
                if (CMP_SUCCESS(rc))
                {
                    lock (stateLock)
                    {
                        processingCapture = true;
                    }

                    UpdateStatus(string.Format("Capture received: {0}", filename));

                    //Save the capture to the local temporary directory (keeping the same file name and extension).
                    string newLocation = System.IO.Path.GetTempPath() + System.IO.Path.GetFileName(filename);

                    UpdateStatus(string.Format("Saving capture to: {0}", newLocation));

                    File.Copy(filename, newLocation, true);

                    // Play the capture. Note that this is invoked in the UI thread.
                    PlayAudio(newLocation);
                    UpdateStatus(string.Format("Capture saved: {0}", newLocation));

                    lock (stateLock)
                    {
                        processingCapture = false;
                    }
                }
            }
        }
