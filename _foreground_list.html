Things have changed in version 2 with the Virtual Device Object. It is now possible to do things that were not possible before. There are two ways that are now supported.

  • Background caching of request
  • State management for foreground/background

The following API will be cached in the background and will be executed the next time the application is in the foreground. Only one request can be outstanding at a time. An error will be returned if a second request of that type is requested in the background.

  • CMPCaptureAudio
  • CMPCapturePicture
  • CMPCaptureVideo
  • CMPHideKeyboard
  • CMPHidePicker
  • CMPSendSMS
  • CMPShowKeyboard
  • CMPShowPicker
  • CMPShowPickerUTF16
  • CMPStartCall
  • CMPTakePicture

These API will be managed by Virtual Device Object and will be automatically updated when the application gains foreground again.
