Channel13 Refers to the underlying communication channel that is established between your application and the shared CMP service by the CMP SDK. The channel is handled automatically in CMP.NET by SessionManager.
Notifications14 Display notifications on the phone. Send notifications using Notifications.
Session15 The session component is used to establish a CMP session (CMP.NET does this automatically for you) as well as control CMP session options. This is done by the SessionManager class.
Capabilities16 Capabilities reporting to figure out what CMP capabilities the client device supports. The various component classes in CMP.NET expose the relevant capabilities for that component.
Controls17 Control of the on-screen Citrix Receiver controls, e.g. to show/hide them. Manage the on-screen controls using ClientControls.
Gvch18 The underlying communications mechanism used by the server to talk to the client (Generic Virtual Channel Host). Generally you should never see GVCH errors as they indicate internal CMP system errors.
VDO19 Virtual device object. The VDO APIs are used to support concurrent apps running in a session that use the Citrix Mobile SDK for Windows Apps.
Capture20 The capture APIs are a set of generic APIs for getting pictures/video/audio from a device.
Emulator21 Emulator. Used by developers to test programs before deploying to Citrix server.
