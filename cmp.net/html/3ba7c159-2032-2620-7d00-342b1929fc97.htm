DpiScaling ClassCitrix Mobility Pack .NET SDK

This class is used to implement DPI scaling in a WPF application. With the ever increasing range of mobile devices (with different screen DPIs), developers need to provide DPI independence for their applications when running on different devices. Without DPI independence apps will have an inconsistent physical size when running on devices with different DPIs.

DeviceDependentSeparatorHeightKey

This resource key returns a double value for the height in DIUs that will draw horizontal separator lines with a consistent physical pixel size. To implement DPI scaling, WPF uses pixel rounding when its device independent values do not end up producing a value that gives an even number of pixels. This means over a series of separator lines some lines will be rounded up while others get rounded down, giving an inconsistent thickness.

Using this property will remove the inconsistent thickness between separator lines, as well as ensure no lines get rounded down to zero pixels. This key attempts to get a thickness as close to one DIU as possible.

Remarks

WPF provides an easy way to implement DPI scaling in conjunction with this class. This class will monitor the DPI of the client device and output a set of scaling factors relative to reference DPI you configured when you initialized the WPF CMP API. You can manually consume the scale values using a ScaleTransform in your application, however it is recommended that you enable the DpiScalingEnabledProperty attached dependency property on your application window(s) instead. This property will automatically attach a scale transform to your window and bind its inputs to the scale values this class provides.

This class will apply an additional +25% zoom when it detects that the user is accessing the application from a tablet. This mimics the behavior of current mobile platforms like Android and iOS. On tablets the standard control sizes are bigger since the extra screen size means you do not need to be so economical with space, and can instead provide controls that are easier to tap with a finger.

This class is not thread safe, you should only call it from your UI thread.
