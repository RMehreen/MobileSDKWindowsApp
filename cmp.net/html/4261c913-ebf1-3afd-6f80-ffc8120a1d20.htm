DisplayBeginLockOrientation Method Citrix Mobility Pack .NET SDK

Note: This API is now obsolete.

Initiates an async request to lock the client device to a specific orientation. Call EndLockOrientation(IAsyncResult) to check the result of the request.

You can use OrientationLockingSupported to check whether the current client supports orientation locking. If it does not support it the operation will throw a NotSupportedException which you will receive when you complete the async operation.

Remarks

It is recommended that you use the newer SetSupportedOrientations(SupportedOrientations) API as it is more flexible. With this older API you can only lock the device to one of the four possible orientations. So if you locked a device to portrait up for example, your user will not have the flexibility to hold the device in either portrait orientation. I.e. if they hold their device in the portrait down orientation the app will appear upside down.

With the new API you can lock to more than one orientation. So you can restrict the device to portrait up and portrait down. This allows the app to auto rotate between either of the portrait orientations without going into landscape.
