KeyboardState ClassCitrix Mobility Pack .NET SDK
This class represents a snapshot of keyboard state. It is used to both request the display of the on-screen keyboard, as well report the current state of the keyboard on the client. When requesting the on-screen keyboard to be displayed, use the public properties with setters to specify the desired keyboard configuration.

AutoCapitalizationMode
The auto-capitalization mode in use by the current keyboard. When requesting the on screen keyboard to be displayed, set this to the desired auto-caps mode otherwise the default of none will be used. Note: not all clients support all modes - check the client capabilities first using IsKeyboardAutoCapitalizeModeSupported(KeyboardAutoCapitalization).
AutoCorrectEnabled
Whether or not auto-correct is enabled. When requesting the on-screen keyboard to be displayed, set this to true to enable auto-correct. Note: not all clients support auto correct. Use IsKeyboardAutoCorrectSupported to determine whether the current client supports it.
ExtendedKeyboardEnabled
Whether or not the extended keyboard is displayed. When requesting the on-screen keyboad to be displayed, set this to true to enable the extended keyboard. The extended keyboard is a user configurable row of extra buttons at the top of the keyboard. The user can add buttons and button combinations like ALT, TAB and CTRL-ALT-DEL to the extended keyboard.
InputControlRect

Used to define the bounding rect of the current input control if you want to receiver to try and keep this rect visible when displaying the on-screen keyboard. This input rect is only used when the input mode is set to panning or scrolling. When the input mode is set to RawMouseInput the receiver will resize the session when the on-screen keyboard is displayed instead (rather than moving it up, above the keyboard). In this case it is up to your application to keep the input field visible in the remaining space available in the session.

Note: this property is never set in the keyboard state reported by GetKeyboardState. It is only ever set if you set it for a show keyboard request.

KeyboardType
The type of the keyboard in use on the client. When requesting the on-screen keyboard to be displayed, set this to the type of keyboard that is desired. Note: not all clients support all types. Use IsKeyboardTypeSupported(KeyboardType) to determine whether the client supports a given keyboard type. When setting this property, requests to set to Unknown or Physical are ignored since they are not valid on-screen keyboard types.
OnScreenKeyboardVisible
Whether or not the on-screen keyboard is currently visible. This is a read only property.
PhysicalKeyboardInUse
Whether or not the physical keyboard is currently in use. This is a read only property.
ReturnKeyText
Defines the text that is displayed on the return key on the on-screen keyboard. When displaying the on-screen keyboard, set this property to change to default text that gets displayed on the return key. Note: not all clients support all of the return key text options. Use IsKeyboardReturnTextSupported(KeyboardReturnKey) to check whether a given option is supported by the client.
