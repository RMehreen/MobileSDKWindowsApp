Display ClassCitrix Mobility Pack .NET SDK
This class holds all the information about the current state of the display of the connected client device, e.g. state such as resolution, DPI and orientation. It also has control functions that affect the state of the display such as locking the orientation and setting the viewport.

CanSetViewport
Used to test whether the current client supports setting the viewport. When this property returns false you cannot use SetViewportOrigin(Int32, Int32) to change the position of the viewport on the client.
CapabilitiesDisplayState

This property returns the initial display state of the client that was provided when the client connected to the session. Since this state is loaded from capabilities it will be up-to-date as soon as the SessionConnected event has been fired.

However capabilities state only changes when a new client connects, while display state can change while a client is connected. E.g. orientation changes trigger the X and Y resolutions to be swapped, or the client may be plugged into an external display device with a different resolution. To get the most up-to-date display state you should use GetDisplayState.

If you call this property when the session is disconnected or if the client does not support CMP, it will return null.

You can test whether the current client supports querying display state using DisplayStateSupported. If the client does not support it, the operation will throw a NotSupportedException which you will receive when you complete the async operation.

BeginGetSupportedOrientations
Initiates an async request to query the current supported orientation state. The supported orientations tell you what orientation locking (if any) is currently applied to the client device. The flags set in the result you get back from the async end routine (EndGetSupportedOrientations(IAsyncResult)) define which orientations the application is locked to. If all flags are set then it means no orientation locking is in effect.

BeginSetSupportedOrientations

Initiates an async request to restrict the client device to a set of orientations. Call EndSetSupportedOrientations(IAsyncResult) to check the result of the request.

The typical usage of this API is to lock the device to portrait or landscape. While you can lock a device to one of the four possible orientations, it is recommended that you always lock to pairs of orientations. So if you want to lock to landscape you should lock to landscape left and landscape right. This allows the user to hold the device in either landscape orientation and the app will auto-rotate between the two, without going into portrait.

You can use OrientationLockingV2Supported to check whether the current client supports orientation locking. If it does not support it the operation will throw a NotSupportedException which you will receive when you complete the async operation.

BeginSetViewportOrigin

When the resolution of the HDX session is bigger than the size of the client's screen the user is able to pan the viewport around on their client. This function allows the hosted app on the server to programmatically alter this viewport position.

This function initiates an async request to set the viewport position. To check the result of the operation call EndSetViewportOrigin(IAsyncResult).

You can use CanSetViewport to check whether the current client supports setting the viewport position. If it does not support it, the operation will throw a NotSupportedException when this async operation is completed.

BeginSetViewportZoomLevel
Initiates an async request to set the viewport zoom percentage in order to zoom the viewport in or out.
BeginUnlockOrientation

You can use OrientationLockingSupported to check whether the current client supports orientation locking. If it does not support it the operation will throw a NotSupportedException which you will receive when you complete the async operation.
