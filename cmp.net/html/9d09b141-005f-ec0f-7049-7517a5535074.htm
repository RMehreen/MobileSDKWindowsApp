ClientKeyboardController ClassCitrix Mobility Pack .NET SDK

Used to control the visibility of the on screen keyboard of the mobile client device running the Citrix Receiver based on the focus of input fields. When this dependency property is activated on Control derived class it will issue a show keyboard control whenever the control gets focus.

You should apply this attached property to any text field the user needs to enter input into. You cannot rely on popping the keyboard up once when you display particular form/control and then hiding it when the form is closed. This is because certain classes of mobile devices (notably Android devices) allow the user to manually hide the keyboad via a physical button on the phone.

Hence if your app doesn't always ask the keyboard to popup whenever an input field gets focus your users can get stuck on input forms with no keyboard displayed.

You can use this class in two ways. You can use it directly in XAML via the dependency properties. You can also call it directly via code using ShowKeyboard and HideKeyboard. The direct calls will only work if you have initialised an instance of the API using CmpApi.

AutoHideKeyboardProperty
Set this property to true to enable the focus based keyboard hide functionality. This will request the client device to hide it's on-screen keyboard whenever the text box loses focus.
AutoShowKeyboardProperty

Set this property to true to enable the focus based keyboard popup functionality. This will request the client device show it's on-screen keyboard whenever the text box gets focus, or the user clicks on it.

Enabling this property also turns on auto-scrolling functionality to keep your text box visible. If you are using LayoutEnabledProperty, then your window will resize when the keyboard gets displayed so it occupies the space above the keyboard. This resize operation can potentially cause the textbox that triggered the keyboard to be displayed to get hidden. The recommended page layout to handle this situation is to wrap the page in a ScrollViewer so the user can always scroll the page to see any portions that get obscured. If you have used this recommended layout then this property will lookup the parent ScrollViewer that contains your textbox. When the resize occurs it will then ensure that the page gets scrolled automatically so that the textbox remains visible.

Note: to hide the keyboard, either enable the AutoHideKeyboard dependency property as well, or manually hide the keyboard at an appropriate time.

KeyboardReturnKeyTextProperty
Used to control the return key text that is displayed on the on-screen keyboard when popping up the keyboard on a UI control using AutoShowKeyboardProperty.
KeyboardTypeProperty
This keyboard type dependency property is used in conjunction with AutoShowKeyboardProperty in order to control the specific type of keyboard that gets popped up. Like that property you must also set this one on TextBox instances. You set the value to one of the enumerated KeyboardType values.
ShowExtendedKeyboardProperty
Use this property in conjunction with AutoShowKeyboardProperty to control whether the extended keyboard is also displayed. By default the extended keyboard is not shown. Setting this property to true will cause the extended keyboard to be displayed (if supported by the client).
