Camera PropertiesCitrix Mobility Pack .NET SDK

The Camera type exposes the following members.

Properties
  NameDescription
CanControlThumbnailSize
Reports whether the client supports the capability to create a thumbnail with a non-default size, i.e. with the ThumbnailType set to one of:
  • Small
  • Medium
  • Large
  • Custom
This only applies to pictures taken with the V2 take picture API: BeginTakePicture2(PictureOptions, TimeSpan, ActionIAsyncResult) or TakePicture2(PictureOptions, TimeSpan).
CanCreateThumbnail
Reports whether the client supports the capability to create thumbnails for pictures taken using V2 take picture requests (BeginTakePicture2(PictureOptions, TimeSpan, ActionIAsyncResult) or TakePicture2(PictureOptions, TimeSpan)).
CanSetDefaultCamera
Reports whether the client supports the capability to set the default camera that will be used to capture the picture for V2 take picture requests. If the default camera can be set, it doesn't prevent the user from swapping to a different camera.
CanSetQuality
Reports whether the client supports the capability to set a custom image quality for V2 picture capture operations using BeginTakePicture2(PictureOptions, TimeSpan, ActionIAsyncResult) or TakePicture2(PictureOptions, TimeSpan).
CanSetResolution
Reports whether the client supports the capability to use a custom resolution for V2 picture capture operations using BeginTakePicture2(PictureOptions, TimeSpan, ActionIAsyncResult) or TakePicture2(PictureOptions, TimeSpan).
CanTakePictures
Whether or not the currently connected client device supports taking pictures using the V1 take picture API.
CanTakePicturesV2
Whether or not the currently connected client device supports taking pictures using the V2 capture picture API. The V2 API adds extra capabilities to control the size and quality of the picture, and returns extra state such as EXIF meta-data and a thumbnail of the image.
