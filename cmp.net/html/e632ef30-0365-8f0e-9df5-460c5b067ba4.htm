ReferenceDpi ClassCitrix Mobility Pack .NET SDK
Used to provide a set of display state that is needed by the DpiScaling class for operation. That most important state it contains, that you must provide is the reference X/Y client DPI that your application is designed for. This is the screen DPI of the primary mobile device you develop on. This provides the reference DPI that is used to implement application DPI scaling. E.g. if you run your app on devices with DPI higher than your reference DPI then the application will get scaled up in proportion.

X
This is the reference X DPI of the client device your application was designed for. When your app is running on devices with this DPI, no DPI scaling will be applied. E.g. if you designed your app to run on an iPhone 3GS you would supply the 3GS DPI of 163. So if you then ran your app on an HTC Desire which has a DPI of 254, your app would be scaled up by a factor of 254/163 = 1.55, in order to maintain the same physical size of your UI elements.

Y DPI equivalent of X.
