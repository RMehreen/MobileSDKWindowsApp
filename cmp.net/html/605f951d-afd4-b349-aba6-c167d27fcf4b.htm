This attached dependency property is used to turn on/off automatic layout for a window. Automatic layout will set the style, size and position of your Window to best match the form factor of the client it is accessed from.

When access from a mobile device your window will have all the window chrome disabled (e.g. close/maximize buttons, resize border) to make it look like a native mobile app. It will also be resized and repositioned so it fits in the available space on the client device. I.e. it handles the following cases that affect screen space:

  • Keyboard popup. Displaying the keyboard will overlay the keyboard over the bottom of the session. This will obscure any content at the bottom of your application. By resizing your app windows to occupy the same dimensions as the receiver client viewport it ensures the keyboard never overlays your application.
  • Orientation changes. When the user flips the orientation between portrait and landscape we resize to fit the new layout.
  • Roaming between devices. Different devices can have different size screens.

When your app is not running on a mobile device, then all the default window chrome will be enabled so you can resize/move/minimize/restore as you would normally do. Your window will also be set to the size controlled by the DefaultWindowSize property. This is set to 1024x768 pixels by default which provides a landscape tablet layout. You can change this to any size you prefer.

Remarks

This layout property uses IsMobileFormFactor to determine whether or not the client device is being accessed from a mobile device. You can query this state yourself by calling GetClientState to get a ClientState instance.

The window style that is applied to your window when it is not being accessed from a mobile client is controlled by FallbackWindowStyleProperty. Typically you will never need to change this property, however there are some cases where you may need to change it to prevent exceptions.
