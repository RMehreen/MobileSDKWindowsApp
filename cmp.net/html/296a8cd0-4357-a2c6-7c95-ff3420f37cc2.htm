ILocationReportT InterfaceCitrix Mobility Pack .NET SDK

This interface is used to expose an instance of a specific type of location report. For each report you can perform a one shot query, or enable events. If you enable events then you will get periodic reporting of location information until you stop the events. For each report you can set and/or query the following state:

  • Accuracy -
  • Minimum interval - When you have location events enabled, you can use the minimum interval to throttle back the frequency at which the report will fire location events. You should set this to the longest interval your app can tolerate in order to reduce the load on processor resources. Note: setting this to a small value doesn't guarantee that you will get events at this interval as the provider is free to provide events at a longer interval if it wants to.
  • Status - This is the current status of the location provider. It is read only.

ReportUpdated

When you enable events using StartEvents this event will get fired to report the latest location. This event firing will continue until you call StopEvents.

You can affect the interval at which the event is fired by using SetShortestInterval(TimeSpan). You should increase this interval to the largest value that your app can use in order to reduce the load on the location provider. While you can specify a small interval, the provider may not respect it. The provider may use a larger interval than the one you supply.

Remarks

This interface exposes both sync and async versions of the APIs. The async versions are useful as many of the calls can take a long time to execute. There are two reasons that the calls can take a long time. Firstly the calls may trigger permission dialogs to be displayed to the user, and so the calls get blocked until the user accepts or rejects the dialog. Secondly the calls involve a request being sent over the network to the Citrix Receiver and a reply being sent back. Depending on network conditions this request/reply sequence can take a long time to complete.

It is recommended that you use the async versions to ensure your user interface does not become unresponsive while the calls are being processed. Keep in mind that the async callbacks may execute in the context of a worker thread.
