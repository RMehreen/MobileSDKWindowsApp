InputBeginShowKeyboard Method Citrix Mobility Pack .NET SDK

Makes an async request to show the keyboard using the supplied state. To check the result of the operation call EndShowKeyboard(IAsyncResult).

You can call IsKeyboardConfigSupported(KeyboardState) to check whether the client supports displaying a keyboard of the requested configuration. If it does not support it, the operation will throw a NotSupportedException when this async operation is completed.

Remarks
If you request a keyboard configuration that the client does not support then your request will get automatically downgraded to a supported configuration so that your request does not fail. E.g. you may request a number pad, however if the device does not support it, it will just show the default keyboard. Keyboard states get downgraded as follows:
  • Keyboard layout (KeyboardType) - Any unsupported layout is downgraded to Standard.
  • Keyboard return key text (KeyboardReturnKey) - An unsupported return key text is downgraded to Default.
  • Keyboard auto-correction - Auto-correction is not supported by any clients, and will always get disabled.
  • Keyboard auto-capitalization (KeyboardAutoCapitalization) - Auto-capitalization is not supported by any clients, and will always get disabled (None).
