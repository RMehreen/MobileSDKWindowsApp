ClientControlsBeginDisable Method Citrix Mobility Pack .NET SDK

Initiates an async request to disable the on-screen Citrix Receiver client controls. To check the result of the request, call EndDisable(IAsyncResult).

This is only supported for clients like iPhone that have permanently accessible controls overlaid over the session area that can get in the way while using apps. For clients like Android the on-screen controls are only shown if you select the physical menu button, so there are no UI elements that get in the way of normal usage of your app. You can test whether the current client supports controlling the on-screen controls using CanControl.

This API supports fallback. This means if you disable or enable the controls when the client does not support this then you will not see an error and your desired state will be cached. If the user then connects via a different client that does support this capability then the cached state will be applied and the client controls will get disabled or enabled as appropriate.
