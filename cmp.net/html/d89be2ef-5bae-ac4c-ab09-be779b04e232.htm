BeginGetShortestInterval

Initiates an async query to get the location events shortest interval.

When location events are active, this is the shortest interval at which the provider may generate events. The provider may take longer to generate events than this interval and it may not raise events on a regular interval and instead wait for the location to change. However it will not generate events more frequently than this value. You should use this interval to throttle back the rate at which you receive events if you do not need frequent updates.

BeginStartEvents

Initiates an async request to start firing location events using ReportUpdated.

The interval at which events get fired will depend on the location provider. Some providers will report events on a periodic interval while others will only fire the event when the location of the device changes. While you can reduce the frequency of reporting using SetShortestInterval(TimeSpan), the provider may always use an interval greater than the one your supply.
