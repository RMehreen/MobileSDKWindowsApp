CmpApi ClassCitrix Mobility Pack .NET SDK

The core class that enables the use of the CMP SDK. You must create an instance of this class before the CMP SDK is usable. You should not create multiple instances of this class in your application, just create one instance and reuse it. You have to create this object in the context of your UI thread. This is so the SDK can get a reference to your UI thread in order to post notifications to it. If you create the SDK in a non-UI thread then none of the notifications or async callbacks will ever get executed.

Many of the API calls in this SDK have both a synchronous and asynchronous version. We provide async APIs where the SDK has to call over the network to the client device. Since the round trip delay can potentially be quite long, you can end up with pauses and stutters in your UI if you use the synchronous version of the APIs. So where available, your should always use the async version of the API instead. This will unblock your UI thread and so keep your UI responsive for the user.

The various functions in this SDK are accessed via the public properties of this class:

  • Audio - Used to capture audio clips using the client device and upload them to the Citrix server so you can use them.
  • Camera - Used to take pictures using the camera on the client device and then upload those pictures to the Citrix server so you can use them.
  • ClientControls - Used to control the appearance of the on-screen Citrix Receiver client controls. E.g. on the iPhone you can disable the on-screen controls that appear at the bottom of the screen.
  • Device - Used to access information about the client device, such as the form factor and client OS.
  • Display - The Display class contains many functions to view and control the state of the clients display. You can query basic information such as resolution, DPI and orientation of the client device. You can also perform control operations such as locking the orientation and setting the position of the viewport on the client.
  • Input - The Input class is used to manage all functionality related to user input. It contains functions to show/hide the on-screen keyboard; change how the receiver processes user touch input and handle button presses of the physical buttons on the user's client device.
  • Messaging - Used to send SMS messages using the phone.
  • Notifications - The Notifications class allows to display notifications to the user using the notification mechanism of the client device, e.g. vibrations, flashing light, text message.
  • Phone - Used to start phone calls.
  • Picker - Used to display a local picker control on the client device. A picker can display a simple list of strings to the user. Since the picker is shown locally on the client, it provides a smoother user experience to the user.
  • SessionManager - The SessionManager class is important for tracking when the user connects and reconnects to the Citrix server. Users may move their session between different client devices. E.g. they may start it on a phone and then reconnect using their tablet. Since the different devices may have different capabilities, you should use the SessionConnected event to detect when the user has reconnected and check the capabilities of the client device in case you want to change the appearance or behaviour of your app.
  • Video - Used to capture video clips using the client device and upload them to the Citrix server so you can use them.

ProcessId

Returns the PID of the process this API instance is associated with. Null is returned when the API is associated with the current process.

When an API instance is associated with another process you can set a subset of SDK state that will be made active whenever that process has focus. This subset is known as Virtual Device Object (VDO) state. The ability to set state on other processes is useful when your application spawns other processes in a session that are not CMP aware. For example you may launch Microsoft Word to view a document. You could then set the touch input mode (SetTouchInputMode(TouchInputMode)) to MouseWheelScroll so swipes trigger scrolling in Word.

The VDO states are:

  • Button redirection - For devices with physical buttons like Android the SDK allows you to redirect the handling of these buttons to your server hosted application. If you want to handle button presses while a non-CMP process has focus you can use an API instance associated with that process to redirect the button and add a handler for it (using BeginSetButtonTarget(ButtonId, ButtonTarget, ActionIAsyncResult) and ButtonPressed). The handler you register will then get called when the redirected button is pressed when that non-CMP process has focus.
  • Client control visibility - Receivers generally have some form of on screen controls. Some receivers allow you to hide these controls so they don't get in the way of using the. However when you hide these controls you need to provide equivalent in app controls where appropriate, e.g. to show the keyboard and close the app. If you want these controls hidden while a non-CMP process has focus you can call BeginDisable(ActionIAsyncResult). Note: this is generally not recommended for non-CMP processes since they will not provide equivalent controls since they are not aware of being accessed from a mobile device.
  • Keyboard state - You can set the keyboard to be displayed when a non-CMP process has focus. Keep in mind that on many devices the user can manually close the keyboard. See BeginShowKeyboard(KeyboardState, ActionIAsyncResult).
  • Orientation locking - You can lock the orientation to a specific orientation (if supported by the device). See BeginSetSupportedOrientations(SupportedOrientations, ActionIAsyncResult).
  • Touch Input Mode - Set via SetTouchInputMode(TouchInputMode), the touch input mode controls how user touch input is translated to events that the server hosted app receives. Since the mode significantly alters the behavior for the user, it is important to ensure your processes use the most appropriate mode. E.g. Citrix Receiver generally uses Pan or MouseWheelScroll for standard Windows apps that are not mobile aware.
  • Viewport position - When the server session size is bigger than the size of the client screen or the user has zoomed the session in while in the Pan input mode you can set the viewport position. To apply a specific position while a non-CMP process has focus you can call BeginSetViewportOrigin(Int32, Int32, ActionIAsyncResult).

Remarks
The SDK creates an internal thread for processing async requests. Depending on your environment you will need to explicitly call Dispose to ensure this thread gets stopped to prevent your app from hanging at shutdown. For WPF apps we can detect app shutdown automatically and will stop the thread for you. However for WinForm apps you must call dispose manually.
Examples
The following example shows how you can issue an async request to send a notification. Note how a callback function has been provided as a parameter. This function will get executed in the context of your UI thread, so you are free to make calls that update your UI in this callback without worrying about throwing an exception.
Copy
CmpApi cmpApi = new CmpApi();
cmpApi.Notifications.BeginNotifyUser(NotificationTypes.Vibrate, null, ar =>
{
    try
    {
        cmpApi.Notifications.EndNotifyUser(ar);
        Trace.WriteLine("The user was notified.");
    }
    catch (CmpException)
    {
    }
    catch (NotSupportedException)
    {
    }
});
