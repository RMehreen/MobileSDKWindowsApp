BeginGetButtonTarget

Initiates an async request to query the redirection state of a button.

When a client connects all buttons will start in the default ClientDevice state.

BeginGetKeyboardState
Initiates an async request to get the current state of the keyboard from the client. Call EndGetKeyboardState(IAsyncResult) to get the result of the request.
BeginGetTouchInputMode
Initiates an async request to get the current touch input mode that specifies how the receiver is translating touch input and passing it up to the Citrix server. Call EndGetTouchInputMode(IAsyncResult) to complete the request and get the result.
BeginHideKeyboard

Initiates an async request to hide the on-screen keyboard. To check the result of the operation call EndHideKeyboard(IAsyncResult).

You can call CanControlOnscreenKeyboard to check whether the current client supports keyboard control. If it does not support it, the operation will throw a NotSupportedException when this async operation is completed.

BeginSetButtonTarget

Initiates an async request to set or cancel button redirection for a button. Call EndSetButtonTarget(IAsyncResult) to get the result of the request.

You can test whether the client supports redirecting a given button using IsButtonRedirectionSupported(ButtonId).

This API supports fallback. This means if you change the redirection state of a button that the client does not support you will not get an error and your request will be cached. If the user then connects via a different client that does support this capability then the cached state will be applied and the button will be redirected appropriately.

BeginSetTouchInputMode
Initiates an async request to set the current connected client to the specified input mode. The input mode alters how the client processes and translates touch events. Call EndSetTouchInputMode(IAsyncResult) to complete the request.
BeginShowKeyboard

Makes an async request to show the keyboard using the supplied state. To check the result of the operation call EndShowKeyboard(IAsyncResult).

You can call IsKeyboardConfigSupported(KeyboardState) to check whether the client supports displaying a keyboard of the requested configuration. If it does not support it, the operation will throw a NotSupportedException when this async operation is completed.
