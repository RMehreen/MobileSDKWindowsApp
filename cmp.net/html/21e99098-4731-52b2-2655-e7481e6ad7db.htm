TouchInputMode EnumerationCitrix Mobility Pack .NET SDK
The input modes define the different ways Citrix receivers will process touch input, particularly swipe gestures. The MouseWheelScroll and Pan modes were mainly designed for interacting with legacy Windows applications, while the RawMouseInput mode is mainly intended for apps designed for mobile use.

Unknown0 The input mode is not known/defined.
MouseWheelScroll1 When in this mode the receiver will generate mouse wheel scroll messages when a user swipes up and down. This allows them to scroll any scrollable areas inside their application. This is most useful for legacy Windows applications.
RawMouseInput2 When in this mode the receiver will treat taps and drags as mouse input. Tapping generates left mouse button messages and dragging generates mouse move messages. This is most useful for applications designed for mobile usage since your app has the freedom to process the raw mouse input in any way it desires.
Pan3 When in this mode the receiver will use swipe gestures to pan viewport around inside the session. This is useful when the session size is bigger than the size of the client's screen and the user is unable to view the entire application. This mode is generally used for legacy Windows applications.
