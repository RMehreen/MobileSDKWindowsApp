CmpApi(Int32)

Initialisation constructor to create an API instance that is associated with another process rather than the current process. You can use this to control Citrix Windows App Mobilization SDK state for processes that do not use the SDK themselves. This means that when the specified process has focus, the state you set using this API object will get synced to the client device. Refer to the ProcessId property for more information.

If you are using CMP in a WinForms or WPF application you should call this from your main UI thread. The API will obtain a reference to the dispatcher (Dispatcher) of your thread and will execute any notifications and async callbacks on this thread.
