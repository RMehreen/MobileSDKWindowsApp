BeginRemovePicture
Initiates an async request to remove the specified picture. Pictures are stored on the client device in the private application data store for the Citrix Receiver. To avoid wasting space on your users' devices you should remove pictures once you have copied them off the client. If you do not explicitly remove pictures then they will get automatically deleted off the device 24hrs after they were created.
BeginTakePicture
Initiates an async request to take a picture. This will bring up the native camera app on the device. The async operation will complete when the user takes a picture or cancels the operation, or the supplied timeout occurs.
BeginTakePicture2

Initiates an async request to take a picture. This will bring up the native camera app on the device. The async operation will complete when the user takes a picture or cancels the operation, or the supplied timeout occurs.

This V2 version of the API adds extra options for controlling the quality of the captured image. Also once a picture has been taken it can return a thumbnail and meta-data about the picture.

RemovePicture
Removes the specified picture from the local storage of the client device. Pictures are stored on the client device in the private application data store for the Citrix Receiver. To avoid wasting space on your users' devices you should remove pictures once you have copied them off the client. If you do not explicitly remove pictures then they will get automatically deleted off the device 24hrs after they were created.
TakePicture
Brings up the native camera app on the device so the user can take a picture. This function will block until the user has taken the picture or cancels the operation, or the supplied timeout occurs. Since this function can block for a long time, it is recommended that you use the async version instead (BeginTakePicture(PictureImageFormat, TimeSpan, ActionIAsyncResult)). If you do call this synchronous version you should not call it from your UI thread, since your UI will lockup until it returns.
TakePicture2

Brings up the native camera app on the device so the user can take a picture. This function will block until the user has taken the picture, or the supplied timeout occurs. Since this function can block for a long time, it is recommended that you use the async version instead (BeginTakePicture2(PictureOptions, TimeSpan, ActionIAsyncResult)). If you do call this synchronous version you should not call it from your UI thread, since your UI will lockup until it returns.

This V2 version of the API adds extra options for controlling the quality of the captured image. Also once a picture has been taken it can return a thumbnail and meta-data about the picture.
