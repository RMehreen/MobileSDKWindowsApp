BeginNotifyUser
Initates an async request to send a notification to the user using one or more of the notification mechanisms supported by the device. You can issue multiple notifications of different types at the same time by setting more than one of the notification type flags. The API supports the following notification types:
  • Audio - An audible beep or sound effect.
  • Light - An externally visible light that is turned on or flashed.
  • Text - A short text message that is displayed to the user. How this is displayed is dependent on the device. It may be some form of popup, or may appear in the notification bar of the device if it supports this.
  • Vibrate - The device vibrates.
EndNotifyUser
Completes a notify user request. Call this to check whether the request was issued successfully.

IsNotificationTypeSupported

Tests whether the supplied notification types are supported. It will return false if one or more of the requested types are not supported by the client.

Note that the supported types may change on a reconnect since the user may have connected using a different device.
